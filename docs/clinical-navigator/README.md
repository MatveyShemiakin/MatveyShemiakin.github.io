# Офтальмологический клинический навигатор

Статус: **исследовательский прототип, не для клинического применения**.

## Назначение

Профессиональный раздел сайта для врачей-офтальмологов должен помогать:

- принимать свободное описание патологии или клинической ситуации;
- выбирать подходящий клинический сценарий;
- сначала проверять настораживающие признаки и срочность;
- структурировать обезличенный клинический случай;
- формировать основные диагностические направления и значимые альтернативы;
- определять недостающие обследования;
- формировать итоговую карточку только по прямому запросу врача;
- в будущем показывать терапевтические варианты только из утверждённой базы;
- объяснять ограничения и указывать источники.

Навигатор не должен ставить окончательный диагноз, самостоятельно придумывать лекарственные схемы или заменять решение врача.

## Текущая версия v0.5

Навигатор переведён из одномодульного демонстрационного сценария в многомодульное клиническое ядро.

Доступны семь рабочих модулей:

1. синдром сухого глаза и дисфункция мейбомиевых желёз;
2. острый красный глаз;
3. кератит и инфильтрат роговицы;
4. вспышки, плавающие помутнения и подозрение на разрыв сетчатки;
5. повышение ВГД, офтальмогипертензия и подозрение на глаукому;
6. передний увеит;
7. снижение зрения после операции по поводу катаракты.

Каждый модуль содержит:

- собственный перечень настораживающих признаков;
- последовательность уточняющих вопросов;
- отдельные правила интерпретации;
- предварительную оценку срочности;
- основные диагностические направления;
- важные альтернативы;
- перечень следующих обследований;
- список недостающих данных;
- итоговую карточку с выгрузкой в TXT или печать/PDF.

## Файлы прототипа

- `/doctors/index.html` — входная страница профессионального раздела;
- `/doctors/doctors.css` — визуальная система раздела;
- `/doctors/navigator/index.html` — многомодульный чат-интерфейс;
- `/doctors/navigator/modules.js` — определения семи клинических модулей;
- `/doctors/navigator/app.js` — маршрутизация, ветвление, интерпретация и карточка;
- `/doctors/navigator/validate-modules.js` — автоматическая проверка структуры модулей;
- `/preview/clinical-navigator/index.html` — постоянный тестовый стенд.

## Ограничения GitHub Pages

Текущий сайт является статическим и размещён на GitHub Pages. В нём нельзя безопасно хранить API-ключ, выполнять серверную проверку рекомендаций или обрабатывать чувствительные данные.

Поэтому production-версия должна состоять из двух частей:

- статический frontend, визуально связанный с основным сайтом;
- отдельный защищённый backend, выполняющий авторизацию, вызовы модели, поиск по базе знаний, клинические правила, журналирование и контроль безопасности.

## Правило публикации

Изменения разрабатываются в отдельной ветке и проходят совместное утверждение. Слияние в `main` и публикация на домене выполняются только после проверки интерфейса, клинической логики и юридических формулировок.

## Следующие этапы

1. Расширить покрытие до основных амбулаторных, неотложных и послеоперационных офтальмологических сценариев.
2. Сформировать формальную матрицу правил и источников для каждого модуля.
3. Создать тестовый набор типичных, атипичных и опасных случаев.
4. Провести независимое клиническое рецензирование модулей.
5. Спроектировать лекарственную базу без свободной генерации дозировок.
6. Подготовить backend API, авторизацию врачей и журнал версий.
7. Подключить языковую модель только после отделения её функций от детерминированного клинического ядра.
